Gi en beskrivelse av hvordan du har brukt CSS-grid og Flexbox-layout, forklar dine valg.

I chose to primarily use CSS-grid because it is a structured layout which was easy to get started with. Flexbox is primarily designed for one-dimensional layouts whereas grid is engineered to create two-dimensional layouts. My gallery required a two-dimensional layout which made it an easy choice.

However, this documentation section is created using flexbox, and it does a great job in creating a one-dimensional layout like this. I am aware that flexbox can also handle two-dimensional layouts, but it is not specifically designed for it like CSS grid is, and it therefore requires a bit more code to make right. What I do like about flexbox though, is the ability to fine-tune the alignment of components to a higher degree than what’s possible with CSS grid.

Gi en beskrivelse av hvordan du har brukt HTML canvas og SVG i HTML.

I have declared all SVG components and corresponding animation transformations in the HTML document, however, the animationTransform elements are missing some parameters which I have chosen to set in a javascript file (svg.js). This is because I wanted to control the animations using javascript as it gives me a bit more freedom and possibilities in state management.

To ensure a responsive design using SVG components I chose to set all the object’s x and y coordinates as well as height and width as a percentage of the height/width of the parent container. The robot always maintains an aspect ratio of 1:0.8 (width:height). Similarly, I have used percentages on all other elements as well to ensure a responsive design throughout the website.

For canvas I simply declared a canvas element in the HTML document and controlled everything using javascript (canvas.js). I am using an eventlistener to listen to clicks on the canvas to start the animation. Animation objects are created using functions that resembles classes with a constructor and custom functions, as they need to be handled a little differently than other objects.

Similar to the SVG components, the width, height and x and y coordinates of the canvas elements are set using percentages of the parent container to ensure a responsive design.

Gi en kort beskrivelse av hvordan du har brukt jQuery

I have used jQuery for simple transformations of SVG objects. The transformation includes hovering over the body of the robot and clicking the robot’s antenna. I made similar transformations with canvas using javascript but my method of creating these transformations proved incompatible with multiple browsers so I removed them for a better user experience.

I also used jQuery to ensure a responsive design by setting relative width/heights of my SVG elements. In addition, jQuery is used for hiding and showing this documentation section.

Gi en kort beskrivelse av hvordan du har utført cross-browser testing

I have tested my website in Chrome, Edge and Firefox and all functionality is compatible. However, as previously mentioned, some functionality implemented with HTML canvas proved incompatible with Edge and Firefox and I chose to remove it. When visiting sources such as W3School and Stackoverflow I made sure to check if the element I was evaluating was compatible with most browsers.

Older versions of IE are known for not supporting animation transformations, and hence my website is not be compatible with their standards, but I chose not to struggle with getting my website IE compatible as it is an outgoing web browser anyway.
